# PRD 文档

本目录存放产品需求类文档。

PRD 的核心目的不是堆叠描述，而是让需求更清楚，让代码更可验证，让错误更早暴露，让重构更安全。

## 适合放入

1. 产品需求说明。
2. 用户故事与使用场景。
3. 功能范围与不做事项。
4. MVP 边界与阶段目标。

## 不适合放入

1. 详细技术实现方案。
2. 测试报告或验收记录。
3. 发布流程或 Git/PR 操作说明。

## 命名建议

使用 `序号-主题.md`，例如 `001-agent协作规范需求.md`。

## 推荐结构

1. `需求名称`：明确功能名称或问题名称。
2. `SMART 目标`：说明本迭代内的具体目标、衡量方式、边界和完成时间。
3. `验收条件`：用可验证的行为结果描述完成状态。
4. `测试类型`：说明需要哪些测试或验证方式保护需求。
5. `完成标准`：明确测试通过、review 通过、CI 通过和风险处理要求。
6. `重构安全要求`：指出哪些核心行为必须被测试保护。
